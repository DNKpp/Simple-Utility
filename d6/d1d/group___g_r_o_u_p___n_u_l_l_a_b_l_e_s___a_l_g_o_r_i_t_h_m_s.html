Detailed Description

Contains nullable algorithm related symbols.

There already exists several pre-defined algorithms, but users may easily write their own algorithm within a few lines.

/* That's the actual algorithm implementation. It simply checks whether the given nullable is invalid (equal to its null object)
* and invokes the given function. Otherwise the nullable is simply forwarded as the result of the function.
* In fact this is an alternative implementation of the ``value_or`` algorithm, where the value is retrieved lazily when needed.
*/
const auto my_lazy_value_or = []<nullable TNullable, std::invocable TFunc>
(
TNullable&& in,
TFunc&& func
) -> std::remove_cvref_t<TNullable>
{
if (in == null_v<TNullable>)
{
return std::invoke(std::forward<TFunc>(func));
}
return std::forward<TNullable>(in);
};
// This factory function isn't requires, but reduces the noise from the calling code.
const auto lazy_value_or = [&]<std::invocable TFunc>(TFunc&& func)
{
// operator >> binds the given parameter to the back (like std::bind_back).
return algorithm_fn{ my_lazy_value_or } >> std::forward<TFunc>(func);
};
const std::optional resultOfValid = std::optional{ 42 }
| lazy_value_or([] { return 1337; });
REQUIRE(resultOfValid == 42);
const std::optional resultOfInvalid = std::optional<int>{}
| lazy_value_or([] { return 1337; });
REQUIRE(resultOfInvalid == 1337);

template<class TFunc >
constexpr auto sl::nullables::and_then ( TFunc &&  func)
constexpr

Passes the value of the input_nullable to the function if it's not equal to its null-object. Returns the null-object otherwise.

Template Parameters
TFuncThe type of the passed function. The function should be invokable with types returned by unwrap and return a type which satisfies the nullable concept (this type does not have to be the same as the type already used in the expression).
Parameters
funcThe given functional object.

This algorithm uses the actual value of a input_nullable and passes it to the given functional. The functional should then return any nullable compatible type as desired. Otherwise, if the input_nullable compares equal to its null-object, the null-object is returned instead.

Note
In the following examples the outcome is always presented within the REQUIRE() statement.

This example shows the outcome when a valid input_nullable is used in a and_then expression. Note that the resulting nullable type differs from the starting one.

const std::optional result = std::optional{ 42 }
| and_then([](const int i) { return std::optional{ std::to_string(i) }; });
REQUIRE(result == "42");

template<std::invocable TFunc>
constexpr auto sl::nullables::or_else ( TFunc &&  func)
constexpr

Returns the nullable if it's not equal to its ''null''-object. Executes the passed function otherwise.

Template Parameters
TFuncThe type of the passed function. The return type must be either `void or such that it can be used to initialize a new nullable object with the same type as used on.

This algorithm simply forwards the nullable if it compares unequal to its null-object. Otherwise the passed function will be called and the returned value initializes a new nullable-object of the same type. If the functions return type is void, the null-object of that nullable-type will be returned instead.

Note
In the following examples the outcome is always presented within the REQUIRE() statement.

This example shows the outcome when a valid nullable is used in a or_else expression.

const std::optional result = std::optional<int>{ 42 }
| or_else([] { return 1337; });
REQUIRE(result == 42);
