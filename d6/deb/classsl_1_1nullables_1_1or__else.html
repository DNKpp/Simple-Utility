Detailed Description

template<std::invocable TFunc>
class sl::nullables::or_else< TFunc >

Returns the nullable if it's not equal to its ''null''-object. Executes the passed function otherwise.

Template Parameters
TFuncThe type of the passed function. The return type must be either `void or such that it can be used to initialize a new object of type std::remove_cvref_t<TNullable>.

This algorithm simply forwards the nullable if it compares unequal to its null-object. Otherwise the passed function will be called and the returned value initializes a new nullable-object of the same type. If the functions return type is void, the null-object of that nullable-type will be returned instead.

Note
In the following examples the outcome is always presented within the REQUIRE() statement.

This example shows what happens when a valid nullable is used in a or_else expression.

namespace sn = sl::nullables;
std::optional<int> opt{ 42 };
const std::optional<int> result = opt | sn::or_else{ [] { return 1337; } };
REQUIRE(result == 42);

This example shows what happens when an invalid nullable is used in a or_else expression and the functional has a return type other than void.

namespace sn = sl::nullables;
std::optional<int> opt{ std::nullopt };
const std::optional<int> result = opt | sn::or_else{ [] { return 1337; } };
REQUIRE(result == 1337);

---

This example shows what happens when an invalid nullable is used in a or_else expression and the functional does not return anything.

namespace sn = sl::nullables;
std::optional<int> opt{ std::nullopt };
const std::optional<int> result = opt | sn::or_else{ [&] { logger << "optional was invalid."; } };
REQUIRE(result == std::nullopt);
