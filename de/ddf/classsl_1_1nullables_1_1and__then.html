Detailed Description

template<class TFunc>
class sl::nullables::and_then< TFunc >

Passes the value of the input_nullable to the function if it's not equal to its null-object. Returns the null-object otherwise.

Template Parameters
TFuncThe type of the passed function. The function should be invokable with types returned by value_unchecked and return a type which satisfies the nullable concept (this type does not have to be the same as the type already used in the expression).

This algorithm uses the actual value of a input_nullable and passes it to the given functional. The functional should then return any nullable compatible type as desired. Otherwise, if the input_nullable compares equal to its null-object, the null-object is returned instead.

Note
In the following examples the outcome is always presented within the REQUIRE() statement.

This example shows what happens when a valid input_nullable is used in a and_then expression.

namespace sn = sl::nullables;
std::optional<int> opt{ 1337 };
const std::optional<std::string> result = opt
| sn::and_then{ toString }; // will be hit, because opt contains a value
REQUIRE(result == "1337");

This example shows what happens when an invalid input_nullable is used in a and_then expression. Note that the resulting nullable type differs from the starting one.

namespace sn = sl::nullables;
std::optional<int> opt{ std::nullopt };
const std::optional<std::string> result = opt
| sn::and_then{ toString }; // won't be hit, because opt is empty
REQUIRE(result == std::nullopt);
