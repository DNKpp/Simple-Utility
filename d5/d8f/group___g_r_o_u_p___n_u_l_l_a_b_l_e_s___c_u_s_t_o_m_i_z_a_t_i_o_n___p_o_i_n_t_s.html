Detailed Description

Contains several customization points, which users may hook for their custom types.

Nullables namespace are designed to work out of the box with types satisfying the corresponding concepts, but sometimes custom types can not be tweaked freely. This is when the pre-defined customization points begin to shine, as users may simply put overloads in the types namespace and let the ADL do the work finding it.

constexpr detail::unwrap_fn sl::nullables::unwrap {}
inlineconstexpr

Retrieves the value of the given input_nullable.

This function is only called if the given input_nullable compares unequal to its null-object, thus it will never access an invalid input_nullable and therefore any additional checks in custom overloads will be unnecessary overhead, which can be avoided.

Note
Custom overloads should never throw.
